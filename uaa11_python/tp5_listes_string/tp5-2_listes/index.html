    Découv 10 : Copie de listes

    Lorsque l'on veut modifier une donnée, il est souvent utile de travailler sur une copie de cette donnée afin de ne pas modifier son contenu initial.

    1. Lis le code suivant. Qu'obtient-on ? 

          nb1 = 5

    nb2 = 3

    2. Exécute le code suivant. Qu'obtient-on ? 

          lst1 = [5]
    lst2 = lst1
    lst2[0] = 3
    print('Contenu de la variable lst1 :', lst1)
    print('Contenu de la variable lst2 :', lst2)

    3. Regarde à présent à l'aide de Python Tutor ce qui se passe lorsque l'on fait une copie de liste avec la méthode classique de l'affection.

    Lors de l'exécution pas à pas du code, on s'aperçoit que la variable lst1 ne contient pas la liste [5], mais un référencement à cette liste (en fait l'adresse mémoire du premier élément de la liste) qui est symbolisé par une flèche (on parle alors de pointeur et on dit que lst1 pointe vers la liste [5]). Ainsi, lors de l'exécution de l'instruction lst2 = lst1, c'est le référencement qui est copié et non la liste. Les deux variables lst1 et lst2 pointent donc aux mêmes endroits comme le montre les deux flèches. Ainsi, toute modification dans l'une des deux variables lst1 ou lst2 affecte les deux variables simultanément.

    Danger

    Lorsque l'on veut effectuer la copie d'une liste, on n'utilise jamais une affectation.
